# Global Claude Instructions

## Communication
- Be concise and direct. Skip preamble and filler.
- Lead with the answer or action, not the reasoning.
- Short sentences preferred over long explanations.

## Code style
- Prefer simple, minimal solutions. Don't over-engineer.
- Don't add comments unless the logic is non-obvious.
- Don't add error handling for cases that can't happen.

## Environment
- Editor: vim
- Shell: zsh
- OS: macOS
- Package manager: Homebrew, asdf
- Infrastructure: AWS (aws-vault), Kubernetes (kubectx)
